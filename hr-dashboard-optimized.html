📋 考勤情况 — 数据口径

指标计算口径数据源
考勤人数当月考勤记录中出现的去重人数DATA.M（月度考勤表）
加班总时长当月所有人员 r[5] 字段求和（单位：小时）M[i][5]（totalOT）
迟到总次数当月所有人员 r[4] 字段求和（单位：次）M[i][4]（lateTimes）
部门加班按 r[2]（部门）分组，累加各人员的加班时长M[i][2] + M[i][5]
加班集中度各部门加班TOP5人员占比 = TOP5加班和 / 部门总加班 × 100%M[i][5] 按人聚合
部门迟到按部门分组，累加各人员的迟到次数M[i][2] + M[i][4]
累计加班/迟到(1-6月)1-6月所有考勤记录对应字段求和遍历 DATA.mM 全月数据

💡 DATA.M 字段结构：[工号, 月份, 部门, 实际出勤天数, 迟到次数, 加班总时长]。月份筛选器决定当月数据，部门筛选器过滤数据范围。

⭐ 绩效考核 — 数据口径

指标计算口径数据源
考核人数当月考核周期中，考核状态 = '已结束' 的人数DATA.P（绩效表）p[7]
绩效等级按 p[5] 字段分类：超出预期 / 符合预期+ / 符合预期 / 符合预期− / 低于预期 / 远低于预期P[i][5]（绩效等级）
占比各等级人数 / 已结束考核总人数 × 100%—
部门绩效分布按 p[3]（部门）分组，统计各部门各等级人数及占比P[i][3] + P[i][5]
健康度评估仅对考核 ≥ 3人的部门评估：
🟢健康：超出预期 ≥ 10% 且 低于预期 < 5%
🔴预警：低于预期 ≥ 15%
🟡关注：低于预期 ≥ 5% 或 超出预期 < 5% 或 符合预期− ≥ 20%
⚪样本不足：考核 < 3人		—
累计考核(1-6月)遍历1-6月所有考核周期，汇总已结束的考核人次DATA.P 全周期
新入职人员绩效近3个月入职（当前月及前2个月）且在职的人员，匹配其当月考核周期的绩效等级DATA.R（入职月份 r[4]）+ DATA.P（考核记录）

💡 DATA.P 字段结构：[工号, 考核周期(一月~六月), 小组, 部门, —, 绩效等级, 员工状态, 考核状态]。
💡 新入职人员绩效口径详解：先找出入职月份落在"当前月+前2月"范围内且状态非离职的人员，再到绩效表中查找其当月考核周期的绩效记录。例如选中6月时，入职月份为4/5/6月的新员工均纳入统计。月份选择器决定考核周期，部门筛选器过滤范围。

🚪 离职预警 — 数据口径

指标计算口径数据源
离职总数1-6月所有状态 = '离职' 的人员总数DATA.R（r[1]==='离职'）
月均离职离职总数 / 有离职记录的月份数（非所有月份）按 r[5]（离职月份）聚合
离职率离职总数 / (当前在职人数 + 离职总数) × 100%
分母 = 统计周期内"总在册人数"		r[1] + r[2]
主动离职占比离职类型 = '合同期内辞职' 的人数 / 离职总数 × 100%r[7]（离职类型）
离职原因分布统计所有离职记录 r[8] 字段的频次分布r[8]（离职原因）
离职类型分布统计所有离职记录 r[7] 字段的频次分布r[7]（离职类型）
各月离职趋势按 r[5]（离职月份）聚合，展示每月离职人数变化r[5]
部门离职人数按 r[2]（部门）分组，统计各部门离职总数r[2]

💡 离职预警看板不受月份筛选器影响，始终统计1-6月全部离职数据。仅部门筛选器生效。
